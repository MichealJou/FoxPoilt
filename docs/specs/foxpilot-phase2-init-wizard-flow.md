# FoxPilot 第二阶段 Init Wizard 交互流

## 1. 文档目的

这份文档用于把第二阶段 `Project Init Wizard` 的交互流程固定下来。

它回答的问题是：

- 用户第一次初始化项目时会看到什么
- 每一步要展示什么信息
- 每一步的按钮和状态怎么命名

## 2. 向导定位

`Project Init Wizard` 不是系统安装页。

系统级基础组合已经在安装阶段处理：

- `Beads`
- `Superpowers`

这个向导负责的是：

- 项目扫描
- Profile 选择
- 平台解析
- 初始化确认
- 初始化完成

## 3. 总体流程图

```mermaid
flowchart LR
  Scan["1 项目扫描"] --> Profile["2 Profile 选择"]
  Profile --> Executor["3 平台解析"]
  Executor --> Summary["4 初始化确认"]
  Summary --> Success["5 初始化完成"]
```

## 4. 步骤清单

### 4.1 第一步：项目扫描

目标：

- 让用户确认当前接管的是哪个项目
- 展示系统已经识别到的项目特征

展示字段：

```text
当前路径
项目名称
项目类型
语言栈
仓库数量
仓库列表
关键扫描信号
```

关键扫描信号示例：

```text
package.json
pnpm-lock.yaml
pyproject.toml
go.mod
Cargo.toml
```

按钮文案：

```text
上一步       不显示
下一步       继续选择 Profile
取消         关闭向导
重新扫描     重新检测项目
```

### 4.2 第二步：Profile 选择

目标：

- 明确这个项目要启用哪种协作方式

候选项：

```text
default         推荐
collaboration   协作优先
minimal         轻量接入
```

每项都要展示：

```text
名称
是否推荐
启用能力
适用场景
```

按钮文案：

```text
上一步       返回项目扫描
下一步       继续平台解析
取消         关闭向导
```

### 4.3 第三步：平台解析

目标：

- 告诉用户系统如何按阶段、角色和平台做解析
- 解释判定依据

展示字段：

```text
阶段清单
角色映射
推荐平台
最终平台
判定来源
判定理由
检测到的平台
是否允许手动覆盖
```

判定来源建议值：

```text
override
project-rule
auto-detect
fallback
```

按钮文案：

```text
上一步       返回 Profile 选择
下一步       查看初始化确认
取消         关闭向导
```

### 4.4 第四步：初始化确认

目标：

- 在真正写配置前，把所有关键决定汇总给用户看

汇总内容：

```text
项目路径
项目类型
仓库数量
选中的 Profile
阶段 / 角色 / 平台 摘要
将启用的能力
将写入的配置文件
```

建议显示的配置文件：

```text
~/.foxpilot/foxpilot.config.json
<project>/.foxpilot/project.json
```

按钮文案：

```text
上一步       返回平台解析
开始初始化   真正执行接管
取消         关闭向导
```

### 4.5 第五步：初始化完成

目标：

- 告诉用户初始化已完成
- 给出下一步推荐动作

展示内容：

```text
初始化结果
项目 ID
仓库数量
Profile
阶段 / 角色 / 平台
下一步推荐动作
```

下一步推荐动作建议：

```text
打开任务中心
查看环境健康
运行 doctor
开始同步 Beads
```

按钮文案：

```text
打开控制台
查看任务中心
完成
```

## 5. 页面状态

### 5.1 加载态

适用于：

- 项目扫描中
- 平台解析中
- 初始化执行中

建议文案：

```text
正在扫描当前项目…
正在解析可用平台…
正在写入初始化配置…
```

### 5.2 错误态

适用于：

- 项目识别失败
- 仓库结构读取失败
- 配置写入失败
- Foundation 未就绪

建议文案风格：

```text
说明发生了什么
说明会影响哪一步
说明用户下一步可以做什么
```

### 5.3 成功态

成功态不只显示“完成”，还要告诉用户系统已经接管了什么。

## 6. 右侧上下文面板建议

右侧面板建议始终存在，用于承接当前步骤说明。

### 项目扫描阶段

显示：

- 当前路径说明
- 项目识别规则说明

### Profile 选择阶段

显示：

- 当前选项差异
- 推荐理由

### 平台解析阶段

显示：

- 阶段 / 角色 / 平台 三层说明
- 平台不是手动 profile 选项的解释

### 初始化确认阶段

显示：

- 将写入的文件
- 变更范围说明

## 7. 设计验收标准

这个向导如果设计正确，用户应该能在不看文档的前提下理解：

- 系统已经做了哪些扫描
- Profile 和平台解析不是一回事
- 系统会先决定阶段与角色，再决定平台
- 初始化后哪些文件会被写入
- 下一步该去哪里继续操作

## 8. 当前结论

`Project Init Wizard` 第二阶段应固定为 5 步：

```text
1  项目扫描
2  Profile 选择
3  平台解析
4  初始化确认
5  初始化完成
```

它是第二阶段最优先的桌面交互入口之一。
