# 任务：FoxPilot Beads API 接入准备

> 2026-03-26 更新：这条线已不再是 FoxPilot 第一阶段的阻塞项。
> 原因见：
> `docs/workspace/2026-03-26-foxpilot-beads-cli-first-integration-decision.md`

## 目标

在本地 `Beads` 快照闭环已经完成的前提下，整理进入真实 `Beads` API 网络同步前必须补齐的外部契约清单，避免后续凭空猜接口。

## 当前前提

当前仓库已经具备：

- `task import-beads`
- `task import-beads --dry-run`
- `task import-beads --close-missing`
- `task beads-summary`
- `task export-beads`

也就是说，本地协议、幂等键、状态映射和安装后命令链路都已经稳定。

## 当前结论

对“当前 shell 可用的本地 CLI 路线”来说，这条线已不再构成阻塞。

原因是 Beads 官方已经明确建议：

- shell 可用时优先使用 `CLI + hooks`
- CLI 本身就是官方主集成路径

因此，FoxPilot 第一阶段不再把“真实 Beads API 网络同步”当成完成门槛。

## 若未来进入远程 / 非 shell 集成，仍需补齐的信息

如果未来产品形态改成“无 shell 环境”或“远程服务化同步”，那时才需要重新打开这份清单。

届时仍然不能直接进入真实网络同步实现，原因不是本地代码底座不足，而是外部契约缺失：

- 未知 `Beads` 的认证方式
- 未知 API 基础地址和环境区分方式
- 未知任务列表接口的分页协议
- 未知增量同步是否支持 `updatedSince`
- 未知“任务已删除 / 已归档”的语义
- 未知字段名是否与当前本地快照协议完全一致
- 未知限流与重试约束

## 下一步只需要补齐的外部信息

### 1. 认证与连接

- 认证方式：`token` / `cookie` / `oauth` / 其他
- 凭证放置方式：环境变量、配置文件还是系统钥匙串
- 是否区分本地、测试、生产环境

### 2. 任务拉取接口

- 获取任务列表的 URL
- 请求方法
- 分页参数
- 排序默认值
- 是否支持按项目或仓库过滤

### 3. 增量同步协议

- 是否支持按更新时间拉取
- 是否支持游标
- 如何识别任务删除、归档或失效
- 如何识别同一任务的稳定主键

### 4. 字段契约

- 外部主键字段名
- 标题字段名
- 状态字段全集
- 优先级字段全集
- 仓库字段或仓库映射字段
- 是否存在额外说明字段可映射到 `description`

### 5. 失败语义

- 限流返回码
- 鉴权失败返回码
- 可重试错误范围
- 单批请求的推荐大小

## 未来打开这条线时的完成标准

- 上述 5 组外部契约有明确答案
- 能据此新增真实 `Beads` 拉取客户端，而不是继续依赖本地 JSON 文件
- 能判断当前 `import-beads` 是否继续复用为“落库层”，还是需要再拆一层网络同步适配器
