# FoxPilot 多语言与路径规范重构设计

## 1. 目标

这次重构解决 4 个问题：

1. 用户可见文案不能继续硬编码为中文
2. 源码导入不能继续使用相对路径
3. 源码缺少可读的字段说明和关键逻辑注释
4. 仓库入口文档需要提供中日英三语版本

本次设计只覆盖 CLI 与文档层，不扩展桌面端。

## 2. 设计结论

### 2.1 多语言机制

- 全局配置新增 `interfaceLanguage`
- 允许值固定为：
  - `zh-CN`
  - `en-US`
  - `ja-JP`
- 默认值为 `zh-CN`
- 所有 CLI 输出和交互文案统一从 `i18n` 消息字典读取
- 首次 `init` 且进入交互模式时，先选择语言
- 后续默认沿用全局配置中的语言
- 提供独立命令设置语言：
  - `foxpilot config set-language --lang <locale>`
  - `fp config set-language --lang <locale>`

### 2.2 路径规范

- 源码和测试中的导入统一改为路径别名
- 禁止继续新增 `../../`、`../` 这种跨目录相对导入
- 统一别名：
  - `@/*` 指向 `src/*`
  - `@tests/*` 指向 `tests/*`
- `tsconfig.json`、`vitest.config.ts` 同步支持该别名

### 2.3 注释规范

- 所有源码文件统一添加标准文件头
- 作者统一为 `michaeljou`
- 导出的类型、接口、函数统一补 TSDoc
- 字段型结构必须补字段说明，尤其是：
  - 全局配置
  - CLI 参数
  - 数据库存储行类型
  - 任务详情和任务目标结构
- 关键逻辑补块注释，重点覆盖：
  - `init` 的补偿顺序
  - 项目解析规则
  - 任务隔离规则
  - 状态更新语义
  - 语言回退逻辑

### 2.4 README 三语

- 根 `README.md` 改为语言入口页
- 新增：
  - `README.zh-CN.md`
  - `README.en.md`
  - `README.ja.md`
- 三份 README 内容结构保持一致
- 根 README 顶部提供切换入口

## 3. 约束

- 代码内部命名保持英文
- 用户可见文案通过语言资源提供
- 注释统一用英文，避免混杂中日英三种注释风格
- 文档三语仅覆盖 README，不要求本轮把全部 `docs/` 三语化

## 4. 分批执行

### 第一批

- 路径别名基础设施
- import 全量重写

### 第二批

- `i18n` 目录
- 语言配置字段
- 语言设置命令

### 第三批

- `init` 首次语言选择
- 现有命令文案迁移到语言资源

### 第四批

- 文件头
- TSDoc
- 字段说明注释

### 第五批

- README 三语
- 根 README 切换入口

## 5. 当前结论

推荐按以上五批顺序推进，不要把路径规范、多语言和文档排版混成一次大修改。先稳住导入和工程配置，再做用户语义层的变更。
